## Legacy v1 Documentation

Root-level package and policy documents describe the v1 module. They remain for
teams maintaining existing v1 applications, but must not be used as the source
of truth for v2 behavior.

- [README.md](README.md) and [README_zh.md](README_zh.md): repository overview
  and v1 usage.
- [RELEASE.md](RELEASE.md), [STABILITY.md](STABILITY.md), and
  [MIGRATION.md](MIGRATION.md): v1 release and compatibility history.
- [MICROGEN_COMPATIBILITY.md](MICROGEN_COMPATIBILITY.md): legacy generated
  output compatibility notes.
- Package README files under the root source tree: v1 package behavior.

Temporary project snapshots, duplicate workflows, and completed roadmaps are
not maintained. Current implementation and release work belongs in the v2
documents listed above.
